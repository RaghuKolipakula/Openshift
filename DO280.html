Red Hat OpenShift Administration II (DO280) Study Guide with Exercises

1. Describe the Red Hat OpenShift Container Platform

Study Material:

  • OpenShift Architecture Components:
    • API Server: Manages REST API calls and handles cluster state.
    • etcd: A distributed key-value store to persist cluster data.
    • Scheduler: Assigns workloads to nodes based on resource availability.
    • Controllers: Ensure the desired state of objects.
  • Differences Between OpenShift and Kubernetes:
    • Built-in security policies.
    • Integrated developer tools and CI/CD capabilities.
    • Enterprise support with Red Hat services.

Exercises:

  1. Explain the function of each OpenShift component.
  2. Compare OpenShift with Kubernetes in a short essay.
  3. Install OpenShift locally using `crc` (CodeReady Containers) and explore basic commands.

2. Verify the Health of a Cluster

Study Material:

  • Installation Methods: Installer-provisioned and user-provisioned infrastructure.
  • Cluster Verification:
    • Use `oc get nodes` to check node status.
    • Monitor pods with `oc get pods --all-namespaces`.
    • Examine logs using `oc logs` and troubleshoot failures.

Exercises:

  1. Deploy an OpenShift cluster using an installer.
  2. Run health checks using `oc get nodes` and `oc get pods -n openshift-cluster-version`.
  3. Review logs using `oc logs` and debug issues.

3. Configure Authentication and Authorization

Study Material:

  • Identity Providers: HTPasswd, LDAP, OAuth.
  • RBAC: Managing users and groups, role assignments.

Exercises:

  1. Set up HTPasswd authentication for users.
  2. Create users and assign roles using `oc adm policy`.
  3. Implement an OAuth identity provider.

By thoroughly studying these topics and completing the exercises, you will be well-prepared to pass the Red Hat Certified Specialist in OpenShift Administration (EX280) exam.
